Overview

Start LaueG by clicking on the desktop icon.

Data processing begins by indexing the spots of a single image, using Single Image mode.

Batch mode is then used to process several image files with minimal user intervention. Normalise is the final stage of batch mode where spot intensities are combined from each image and corrected for wavelength effects and other factors. The final results are .cif and .hkl files that can be used by standard crystallography programs.

Single Image and Batch modes are started from the Scilab Console window. When an image file (or multiple files for batch mode) is selected, LaueG will change the working directory to the file’s location. When you restart LaueG it will return to the last working directory.

HINT: Use a separate directory for each data-set.

HINT: Tools → Rename Image Files can be used to easily rename multiple image files.

Single Image Mode

Use Start→Single Image Mode from the Scilab Console window to display the Image window.

File Menu

The File menu allows loading a new image file, load/save the index file, take a snapshot of the image window, and display relevant information.

Display Menu

The earlier image window showed the raw image. It is usual to show the image with the background removed using Strip Background.

Display options are saved for the current working directory and reloaded on return to this directory. The complete list of options are:

Lighter Image brightens the image.

Darker Image darkens the image.

Brightness Levels enables you to manually change the brightness based on the intensities seen. In this window clicking Test enables you to view the change without making the change. Click Cancel to clear any changes or Accept to adjust the image.

Invert Grey levels changes between dark spots on a light background, or the reverse.

Strip Background removes the background from the image.

Reload Background enables you to see the image complete with the background if desired.

Blur Main Image is useful for very weak images where background noise is significant.

Remove Blur removes the blurring.

Below is the result of Invert Grey levels and Strip Background.

LaueG uses the mouse-right-click to create a Zoom window of the area near the mouse pointer. The enlarged image show the image data in pixel-by-pixel rather than the averaged over 4 x 4 pixels for the main image. The Zoom window is removed by a second right-click.

A second zoom method uses the mouse-scroll-wheel which enlarges the images drawn in the Zoom and main Image windows. A left-click can also be used to move the centre of the enlarged view.

Index Menu

For Quasi-Laue instruments IMAGINE and LADI-3 refer to Appendix 4.

The Index menu is used to index the image (allocate HKLs to each spot), refine cell dimensions and instrument parameters, and to transform the unit cell or change the cell centering.

 

The first step is Input Unit Cell to input the crystal system, centering and cell dimensions. The spacegroup is not needed for LaueG.

The spots are indexed using Index Spots. The image will display orange diamonds and blue circles corresponding to the observed and calculated spot positions. The obs. and calc. spots will not exactly match, as the crystal orientation and cell dimensions have not been refined. As the spots are now indexed, LaueG displays a “guess” of the HKL for the mouse pointer position below the image.

The orientation of the crystal can now be refined to improve the matching of orange and blue boxes. Do this using Refine orientation. An output box will be displayed showing the number of matched spots to the number of observed spots. Ideally the rms (root mean square deviation) number should be below 0.3. To check convergence, repeat Refine orientation.

You can save the current index using File→Save Index File, though this is not necessary if you proceed directly to Batch mode. You can also transform the lattice or change the centering using Transform Lattice. Use the Test button to see the effects of the changes on the image.

HINT: Reset Orientation is useful when the software get confused after a series of unsuccessful indexes.

 

Obs. Spots Menu

Show Obs. Spots Change how many observed spots (blue circles) are displayed on the image. This is particularly useful for images with poor statistics to remove spots that are not real.

Hide Obs. Spots Hides all of the observed the spots. Use Show … to get them back.

The following options are for more advanced use of LaueG.

Spot Search Perform a search for observed spots using a given min. spot separation.

Prune Spots Reduce number of observed spots in the stored list.

Add/Remove Spot (left click) Left-click adds/removes obs. spots to/from the stored list.

Recenter Spot (left click) Recenters an obs. spot that is close to the mouse pointer.

Recenter All Spots Recenters all obs. spots in the stored list.

Calc. Spots Menu

Show Calc. Spots enables you to change the number of spots calculated by the unit cell input by either changing the minimum value of the d-spacing or the minimum wavelength observed. Test enables the user to view changes before accepting them.

Hide Calc. Spots Hides all calculated spots. Use Show … to get them back.

Find HKL Spot Hides all calculated spots. Use Show … to get them back.

Show Unmatched Spots Display as purple circles all observed spots not close to a calculated spot.

Hide Unmatched Spots Hides all unmatched spots. Use Show … to get them back.

The most important of these functions is Show Calc. Spots which defines the limits of calculated spots displayed on the image. For the example image there are ~10 obvious spots near the center of the image that are not matched to the calculated spots. Index2 Menu

More advanced options for indexing/orientating a crystal

Index from 3 Spots Creates a UB matrix using three HKL, spots positions, and wavelength.

Manual Index Spots Allows control over parameters used to index the spots.

Manual Refine Orient Allows manual control over parameters to refine the orientation.

Manual Cell Rotation Can arbitrarily rotate the lattice and shift the X,Y image center. It can also rotate the lattice about a pivot point, in this case, a rotation of 2 degrees about the high-symmetry spot on the right of the image.

Input Lauegen Angles For old-timers that like to use the Lauegen software.

HINT: Before using these functions, try using the simplified indexing of Index Menu on different images, and use Reset Orientation between each index attempt.

Conics Menu

“Experimental” functions using Laue Conics. (Can be useful)

Mark/Unmark Conic Spot (left click) Use left click to mark (blue crosses) observed spots (blue circles) on a conic (the curved lines of spots that pass through the image center). After two or more spots have been marked a conic line (blue dashed line) is drawn through them. All spots on a conic line share a plane of reciprocal space (i.e. the conic spots 012 and 100 make a conic line containing all (h,k,2k) spots).

In the image above I have marked two conic spots to investigate the unmatched spots mentioned in the Calc. Spots Menu section. Directly below the image center we see two unmatched spots,  the weak one on the left is real while the stronger one to the right could not be matched to any conic, that is it is from a different lattice. Using this method I decided that all of the stronger unmatched spots were from a small crystallite misoriented  with respect to the main crystal.

Nodal spots are the high-symmetry points where several conic lines cross. By marking three nodal points it is possible to index a lattice ab-initio. This part is experimental and difficult to use. Please contact me (rop@ansto.gov.au) if you really need to use it. However, I would suggest it is easier to get a quick X-ray determination of the lattice.

Twins Menu

“Experimental” functions for twinned crystals. (Not fully implemented)

For twinned crystals LaueG uses three lattices: the “default” lattice which is used for non-twinned crystals; separate lattices for twins 1 and 2. More than 2 twins is not implemented. This menu is used to copy lattices between the default and the twin lattices. When twinned spots are integrated (in Batch mode) the default lattice is ignored.

At the moment (25-Jun-2018) you should be able to use all these functions, but I am not happy with the Refine Twin Orientations. I also believe that twinned spots can be integrated in Batch mode, but the there are problems with normalising the intensities. Contact me (rop@ansto.gov.au)  for an update on processing twinned data.

Modulated Menu

“Experimental” functions for modulated lattices. (I think they work, maybe)

I believe these function work, but I am not certain that they do. I have a modulated structure to analyse for a user, so these functions will be tested with a real sample in the near future.

Batch Mode

Next all the images in the data collection can be orientated using the index file from the first image. To do this in the Scilab Console go to Start→ Start Batch Mode. Select the image files (usually all of them) you wish to analyse and click open.  When in Batch mode the Image window will disappear and the Orient, Integrate, and Normalise menus on the console window will become useable.

HINT: All image files must be in the same directory and must have names with a common first part followed by a number, such as, D_001.tif, D_002.tif, … , D_015.tif or Ag_Pyd1, …,  Ag_Pyd9, Ag_Pyd10, … , Ag_Pyd18. Use the Tools→Rename Image Files function from the Console window to easily rename multiple files.

Orient Menu

These functions are used to refine the orientation of all Batch mode files without having to use Single Image mode on each file.

 

Load Index File can be used to load an index solution from a file into memory. If you have just been using Single Image mode then this step is not needed as the last index solution will already be in memory. The function Refine Orientation (initial) is used to refine all image files starting from the index solution in memory. This function stores the refined orientations in (*.idx) index files. If you wish to repeat the refinement you should use Refine Orientation (repeat) which reads in the index file for each image, refines the orientation, and save the new orientation in the index file.

For both Refine Orientation functions the dialogue below appears in the console window. This process is slow, so wait until you see FINISH: Orient Spots.

These results are typical for a crystal with a medium sized unit cell. The important point is the Final values are around 0.4 or better and the Cell Dimensions are consistent.

Explanation of the Dialogue

File – displays the file name of the image

Observed – The total number of spots observed.

Matched – The number of observed spots that the software matched to calculated spots. A sudden drop is this value should be investigated.

Start – The starting r.m.s. value in mm between the observed and calculated spots.

Final –The final r.m.s. value in mm between the observed and calculated spots. For small crystal this should ideally be below 0.2 mm, and for larger crystals half the size of the crystal. Values above 1 mm are probably wrong and should be investigated.

Cell dimensions – shows the working a,b,c,α,β, for each image orientation. This is refined by the orientation programme. It should remain fairly constant, though this is not always an indicator that everything is OK. The calculated cell dimensions are very inaccurate and should never be used as true values.

HINT: Sometimes the initial orientation refinement does not work properly, often due to the crystal rotating slightly between exposures, but it also may be due to poor intensity statistics, poor quality crystal, or an instrument or sample environment fault. Below I have simulated a crystal rotation between exposures py8_4k_6 and py8_4k_7. Note the big drop in matched spots and the increase in r.m.s. values, even though the cell dimensions don’t look too bad. After py8_4k_7 the incorrect orientation is propagated to the later images as the initial orientation refinement uses the orientation from the last refinement.  The correct orientation may be found after refining a few more images, but then maybe it doesn’t. The solution is to use Single Image mode to index the first offending image, and then start Batch mode with the files starting from this image. Refine Orientation (initial) will then use this new orientation to refine the later images.

Integrate Spots Menu

The next operation is integrating the spots for all batch files. Two options exist, running a new or old version of the integration program, Argonne_Boxes. In both cases a popup window appears allowing the user to change the settings. Below is popup for the new version used in Integrate spots.

More information on these parameters and the integration method are given in Appendices 1 & 2.

HINT: For the initial integration the only value to worry about is the Min. d-spacing of hkls. For a small unit cell crystal of high quality, the number may be as small as 0.45. For a typical solution grown organic with a medium unit cell, a value closer to 0.8 is common. After the initial refinement we will use Intensity vs d-spacing to obtain a better estimate of the value to use.

 Once the integration starts the Console window will show the progress. Let it run until FINISH: Integrate Spots.

Explanation of the Dialogue

File – The image file name without the .tif ending

Model – The number of “nice” spots in the image used to model the spot shapes

Pass – Total number of spots that were successfully integrated

Full – Strong spots without outer ellipse overlaps

Weak – Weaker spot without outer ellipse overlaps

PkPk – Spots with overlapping outer (peak) ellipses

CoPk – Spots with overlapping outer ellipse and inner (core) ellipses

CoCo – Spots with overlapping inner ellipses

Twins – Number of twins where spots strongly overlap

Fail – Total number of spots that could not be integrated

Edge – Spots that are too close to the image edge, central hole, or known scratches

Neigh – Spots that are too close to a strong neighbour spot

Back – Spots have a strongly varying background

Pixl – Pixel overload detected in spot

Far – Observed and calculated spot centres are too far apart

The Integration→Intensity vs d-spacing function is used after an initial integration to decide on the best value for the minimum d-spacing. A table will appear in the Console window summarising the intensities obtained by integration of the batch files.

The top rows are the most important and show the statistics for the shortest d-spacing data. Usually you try for 1% of spots with >9 esd (Intensity >  9 ´ error) for the shortest d-spacing spots. For the above example we could try reducing the d-spacing a little to try to get more useful spots. For large unit cells you should also take into account the number of Fails which increases with smaller d-spacings due to the increased overlap of calculated spots.

After the integration, Display Ellipses can be used to check the integration process. An image window will appear containing ellipses drawn for each integrated spot. The ellipse colours are based on a cold-hot scale with the blue ellipses showing the high quality Full spots, the green-yellow-orange spots show integrated spots where spot overlap is becoming increasingly bad, and then the red ellipses show the unsuccessful Fail spots.

Each spot shows two ellipses, an inner and outer, as used by Argonne_Boxes to integrate the intensities. For weak spots the intensities within the inner ellipse are summed, and for the strong peaks the intensities within the both ellipses are summed. The Ellipses→Model Ellipses function of the image window will display the model spots used for by Argonne_Boxes. These spots should be spread across the whole image. For more details on Argonne_Boxes see Appendix 1 and the references.

Use the Files menu of the image window to display the integration for other image files.

Check Outliers Menu

Click on Normalise→Check Outlier Spots to view the individual outlier spots and their ellipses. Options to change the display can be found in the Display menu.

Figure 17 - Check Outliers display

Spots can be rejected for many reasons. The spot is off centre in the ellipses, the background is uneven, a scratch of mark on the detector, a strong spot within the first two ellipses of the target spot, a contamination spot from a second crystal in the sample, and many others. To select spot to reject by clicking on them. If there are more than 50 spots use Rejects→Next and use Rejects→Previous to inspect all reflection. Finally use Rejects→Add Marked to Rejects to store the marked spots to be rejected in all future normalisations.

Hint: To reject all spots, do not click on any spots then use Rejects→Add Unmarked to Rejects.

Sometimes the outlier is correct and several of its equivalents are the true outliers. You can inspect the outlier with all of its equivalent reflection using Normalise→Check Outliers + Equivs. Be warned, for crystals with high symmetry this can results in hundreds of spots to inspect.

In both these modes by scrolling over a particular spot you can view information about the reflection associated with it, its associated wavelength, its x and y coordinate in the image and which image the spot has come from. This is displayed at the bottom of the window.

After the rejected spots have been found, normalise the data again. This is generally an iterative process of two or three times, depending on the quality of the data.

Updating the wavelength profile

Use Normalise→Load Wavelength File and select “New wavelength file created by Laue4” to load the newly created wavelength profile. You can view the changes in the wavelength profile using Normalise→Plot Wavelength File. The newly created profile often has several dips and peaks not present in the original profile. This is not a problem, however if the overall shape of the two profiles differs significantly this may indicate you are using the wrong cell, or (in rare cases) that the sample is strongly absorbing.

Iterating the Normalisation procedure

Does rejecting the intensity outliers improve the R(Merg) values? If not, try turning on the extinction or harmonics corrections. These corrections are most common for high quality inorganic crystals with small unit cells.

The wavelength profile is refined by the normalisation software and should be updating after a few iterations.

More detailed statistics and information is available in the file “laue4.lis” created by the normalisation software. In particular the “Comparison of observed and equivalent intensities” table shows intensity statistics binned against several experimental quantities which can show problems with the data or the model used for normalisation.

Appendix 1: Spot Integration Method

Below we show an idealised peak in 1D. Most of the peak intensity is inside the dark green area, the rest of the peak intensity is in the light green area, and outside these areas the intensity is just the background which is fitted by a straight line. The Lehmann-Larsen[4] method  of integrating 1D peaks uses two passes. In the first pass the strongest peaks (model peaks) are used to calculate the peak widths and to measure the ratio (the peak fraction) of the peak intensity in the dark green area compared to that in the light and dark green areas. In the second pass the strong peaks sum the full full intensity in both areas, while the weak peaks only sum the inner (core)  area and then use the peak fraction to estimate the intensity missed in the outer (peak)  area.

To determine the width of the peak we “cut” the peak at a fraction of its height above the background. In the image below the cutoff level has been set to 0.5. The software then scales this width so that the peak fraction is equal to a target value. This gives the width for the core area, and the width of the peak area is assumed to be some multiple of the core width.

The integration method of Wilkinson et al.[2] extends this method to 2D and 3D data  created using area-detectors. For our 2D Laue data we replace the width of a peak by three parameters used to define the shape of an ellipse.

Appendix 2: Integration Parameters

Target peak fraction – Ratio of 0 to 1 used to calculate the size of the core ellipse.

Area multiplier for peak ellipse – Ratio of the areas of the peak ellipse to the  core ellipse.

Peak fraction uncertainty  – The percentage uncertainty in the peak fraction correction.

Neighbour overlap tolerance – A value of 0 means any overlap with a neighbouring spot results in the spot being ignored, while a value of 100 effectively means this never happens. Values of 1 and 10 correspond to conservative and questionable compromises on overlap.

Model ellipse in middle zone – The target number of model peaks for the central part of the image covering 1/3 of X and Y.

 Model ellipse in outer zones – The target number of model peaks to use in each of the four outer areas surrounding the middle zone.

Cutoff level of contour ellipse – Ratio of peak height used to determine contour ellipse.

Min. fill fraction for contour – The mininum fraction of pixels inside the contour ellipse that are above the cutoff level.

Recenter strong peaks – Toggle this box to recenter the strong peaks before integration.

Min. d-spacing for hkls – The minimum d-spacing used to generate the spots that are integrated.

Min. wavelength for hkls – The minimum wavelength used to generate the spots that are integrated.

Max. wavelength for hkls – The maximum wavelength used to generate the spots that are integrated. Only used for quasi-Laue instruments with a restricted wavelength range.

Twin Mode and Modulated Mode – These options are available integration of of twinned or modulated lattices. If not selected the “default” lattice will be integrated instead.

HINTS:

Setting optimal integration parameters requires expert knowledge that is difficult to convey in a document. It requires the use of several sets of parameters, followed by separate integration, normalisation, and model refinements to determine which give the best final results. Optimal parameters could result in a 10-50% reduction in the esds of atomic parameters, but most likely it will be about 20% for quite a lot of effort. Fortunately, in most cases the default values give an adequate solution for most users. However, for weakly scattering crystals and/or those with poor spot shapes the default parameters may not be suitable and changing the integration parameters becomes the difference between a successful or failed experiment. Details of the integration process for each image file is written to a *.lis file. These files were designed for my own use and are extremely verbose, but the advanced user may find them of some use in optimising integration parameters.

The Target peak fraction, Area multiplier for peak ellipse, and  Peak fraction uncertainty parameters are generally changed together. The first defines the size of the core ellipse, the second the size of the peak elllipse, and the third determines the uncertainty in estimating the intensity between the core and peak ellipses. For images where the spot shapes are approximated well by ellipses the values of 0.8, 3.0, 25 are commonly used as a conservative values. The statistics of the weak spots can be improved by reducing the core ellipse (possibly using 0.5, 6.0, 50) but this increases the possibility of systematic errors in the intensities. For poor spot shapes the ellipses can be increased in size using 0.9, 4.0, 50 to ensure most of the intensity is contained within the ellipse areas.

If the number of model spots is much less than the target values you should try changing the Cutoff level of contour ellipse and Min. fill fraction for contour parameters. The software creates a list of possible model spots and removes the weakest peaks in the list until it get the target number for each zone of the image. However, spots can be rejected before entering the list and the above two parameters can reduce the number rejected. The min. fill fraction can be reduced from 0.8 to 0.5 so to accept lower quality spots as models. Increasing the cutoff level from 0.1 to 0.5 will also reduce the number of spots rejected as a cutoff at 50% is much higher above the background level and so easier for the software to determine the spot shape. Unfortunately, many crystals have spot shapes that vary with the cutoff level  and lower cutoff levels give spot shapes that are better to describe the shape of the spot between the core and peak ellipses.

I believe in most cases the Recenter strong peaks option should not be used. If it is, you should compare the final results with this option on and off to decide which is better.

Min. d-spacing for hkls and Min. wavelength for hkls are used to define how many spots are generated for integration. Values too small result in more spots  with effectively zero intensity and increased number of integration failures due to spot overlap. Values too large result in too few spots with spots of significant intensity not being integrated and not contributing to the final merged intensities. Both values are dependent on crystal quality and unit cell size, though min. d-spacing to a much greater extent than min. wavelength. For a small unit cell crystal of high quality, values could  be as small as 0.45 and 0.80. For a typical solution grown organic with a medium unit cell, values of 0.8 and 0.90 are common.

Appendix 4: Indexing for Quasi-Laue Instruments

Quasi-Laue data requires a slightly different indexing method. For true Laue data the high-symmetry “nodal” spots are generally the most intense spots and the automatic spot search can easily identify them. For Quasi-Laue data a short list of nodal spots (about 10) must be entered by hand to index the image. Once this is completed a new “spot search” must be performed so that a large number of spots are available for the “refine orientation” process.

NB: Older versions of LaueG do not have a Spots pulldown menu. Instead you will find the same menu items in the Index 2 pulldown menu.

1)      Small blue circles on the image indicate you already have some spots in the spots list. You must erase them using Spots→Clear Spots List.

2)      Manually add the nodal spots using Spots→Add/Remove Spot.  It is often useful to zoom in using the right-click before clicking on a spot.

3)      Once you have ~10 nodal spots, optimise their positions using Spots→Recenter All Spots.

4)      Use Index→Index Spots to index the image as described in Index Menu. If the indexing fails, try adding or removing nodal spots using the menu items in Spots. If you still have no success, try repeating the process on a different image.

NB: Do not use Index 2→Index Spots to index the spots!

5)      Once you have an index, you need to do a spot search using Spots→Spot Search. This should produce hundreds of small blue spots around the found spots.

6)      You can now use Index→Refine Orientation and proceed as described in Index Menu.

Software Problems?

Q: Why are the Mac and Linux versions so out of date?

A: Due to lack of interest! Email me at ross.piltz@ansto.gov.au if you really need one.

Q: How do you start LaueG? Where is the LaueG icon?

A: In some cases the icon is there but has a blank image until Windows is restarted. In other cases no icon labelled “LaueG” can be found on the desktop. In that case, copy the icon file from the installation files onto the desktop, then right-click to change its properties to point to the correct location for your installation of SCILAB.

Q: The displayed image rotates in 3D when I right-click and move the mouse.

A: You are running an unpatched or incompatible version of SCILAB. Reinstall LaueG to install the necessary patches for SCILAB and check the installation comments to make sure you are using a compatible version of SCILAB.
